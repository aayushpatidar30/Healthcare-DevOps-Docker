<h2>Problem Statement</h2>
<p>Containerize a Healthcare service (built with Python Flask) using industry best practices for Docker. The project includes writing multi-stage Dockerfiles to optimize image size and security,
 alongside a Docker Compose configuration to orchestrate the application and its database dependencies. This ensures a consistent, portable development and production environment across the entire team.</p>
 <hr>
 <h2>Overview</h2>
 <p>This project focuses on containerizing a Healthcare Management web application built using Python Flask with Docker. The system enables patients to register, log in, book appointments, and access medical reports, while doctors can manage patient information and update reports through a web interface.

The emphasis is on:
<ul>

<li>Docker-based containerization</li>

<li>Consistent and portable environment setup</li>

<li>Secure database connectivity</li>

<li>Scalable deployment using Docker Compose</li>

<li>Easy application management and maintenance</li>
</ul>

The main objective is to automate deployment, reduce configuration errors, and provide a reliable, reproducible healthcare service that can run on any system without complex manual setup.</p>
<hr>
<h2>Objective</h2>
<ul>
<li>Containerize the Flask healthcare application using Docker</li>

<li>Ensure consistent and portable deployment environment</li>

<li>Manage database and services using Docker Compose</li>
